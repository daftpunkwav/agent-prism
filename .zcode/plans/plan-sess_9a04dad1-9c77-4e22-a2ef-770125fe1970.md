### 关键设计决策

1. **沙箱**：保留 Python 内置白名单方案（不引入 `RestrictedPython` 等新依赖，遵循 CLAUDE.md "优先用标准库"），但 **去掉所有可反射的入口**——`type / Exception / isinstance / True / False / None` 等会从 whitelist 中移除，仅留纯函数 (`len/range/print/...`)；超时改为 `multiprocessing.Process` + `terminate` 以真正可中断。
2. **持久化原子写**：抽出 `_atomic_write_json(path, payload)` 帮助函数（tmp + `os.replace`），`config.py` 与 `project.py` 复用；旧文件 `.bak` 备份。
3. **SSE 取消**：`stream_parallel` 改成异步生成器，**自身接收 `asyncio.CancelledError` 时主动 `task.cancel()` 所有 worker**，并在 worker 的 `try/finally` 中调用 `get_workspace_mgr().remove(ws_name)` 清理内存 dict。
4. **OpenAI 兼容**：`create_chat_model` 按 `api_format` 分发；`requirements.txt` 增加 `langchain-openai`；`test_provider` 同样按格式分发。
5. **Workspace name 暴露**：`ArenaEvent` 增加 `workspace: str` 字段（仅 `complete` / `token_update` 阶段），由 adapter 写回；前端 `activeWorkspace` 直接消费。
6. **`api_key: ""`**：前端 `saveProvider` 不再传 `api_key` 字段（let backend keep-existing 逻辑生效），保留 UI "留空则保持" 文案；后端 `ProviderConfigUpdate.api_key` 加 `Field(default="")` 保留兼容。
7. **类型补强**：`Project.results` 用新增 `PipelineRunResult` Pydantic 模型；`workspace_files` 用 `dict[str, WorkspaceFileEntry]`；`metrics_summary` 用 `dict[str, PipelineMetrics]`。
8. **事件类型**：在 `EventType` Literal 中加入 `"harness_edit"`；前端 `ArenaEvent` 改为判别联合；`api.ts` 的 `streamArenaRun` 增加 `event:` 字段处理、`[DONE]` sentinel、解析错误传播。
9. **AbortController**：每个 fetch 用 `AbortController`，effect cleanup `[]` 一次 + `[workspaceName]` 一次双管齐下。
10. **a11y**：所有 icon-only 按钮加 `aria-label`；Tab 列表加 `role="tablist"` / `role="tab"` / `aria-selected` + 键盘左右切换。

### 风险与回退

- **风险**：sandbox 收紧后部分历史多轮交互可能不再工作（仅当 LLM 真的在调用 reflection chain 涉及 `type()`）。
  - **回退**：保留旧 `_safe_run_code` 为 `_safe_run_code_legacy`，通过环境变量 `SANDBOX_MODE=legacy` 切换（默认新实现）。
- **风险**：把 `api_key` 从 `saveProvider` body 移除后，旧版前端调用方式 `saveProvider({ ...config, api_key: "" })` 仍然不会丢 key（后端 keep-existing 兼容），但语义需在 settings 页 UI 中明示。
- **风险**：`SSE worker cancellation` 改动可能影响 e2e；通过现有 `test_runner.py`（如缺则新增）覆盖 cancel 路径。
